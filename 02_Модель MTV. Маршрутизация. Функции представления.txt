создадим первое приложения сайта - ядро сайта

cd coolsite
python manage.py startapp women    (women - просто имя нашего приложения)

появился каталог(пакет) women

теперь нужно зарегистрировать в проекте
открываем в папке coolsite файл settings.py
дописываем в INSTALLED_APPS название нашего приложения 'women' так будет работать но лучше так => 'women.apps.WomenConfig'
на деле 'women' => файл приложение 'apps.py' => и класс class WomenConfig(AppConfig)
в итоге прописывем полную строку чтоб не приходилось постоянно поправлять сссылку на класс, и упростить работу с классом конфигурации
        'women.apps.WomenConfig'

создадим обработчик главгой страницы сайта, для этого нужно определить представление этой страницы ( в виде функций или классов)

пердставления прописываем в файле views.py
def index(request): # HttpRequest (index - проссто имя и может быть любым)
    return HttpResponse("Страница приложения women.")

далее нужно связать функции с соответствующим URL адресом
в пакете coolsite в файле  url.py добавляем в список urlpatterns наш маршрут path('women/', index),
нужно импортировать имя index -> from women.views import index
дальше, если выдаст ошибку то coolsite правой кнопкой внизу Mark Directory as => Sources Root и должно заработать

и теперь по ссылке http://127.0.0.1:8000/women/ мы видим надпись  "Страница приложения women."

__
для примера добавим в views.py функцию списка статей по категориям
def categories(request): # HttpRequest (index - проссто имя и может быть любым)
    return HttpResponse("<h1>Статьи по категориям</h1>")
пропишем ее URL  в файле urls.py в список urlpatterns
path('categories/', categories),   # http://127.0.0.1:8000/categories/
и исправим импорт чтоб все имена подключались-> from women.views import *

__
теперь по этому пути http://127.0.0.1:8000/ выдает ошибку
мы можем сделать чтоб по этому пути отображалась наша главная страница
изеним строку(маршрут) в файле urls.py  path('women/', index) на  path('', index)
и теперь по http://127.0.0.1:8000/ отображается  "Страница приложения women."

!!! ПРОПИСЫВАТЬ МАРШРУТЫ ТАКИМ ОБРАЗОМ не очень хорошо в Django есть другой способ
в файле urls.py импортируем  from django.urls import path, include
прописываем маршрут
path('women/', include('women.urls')), # в файле women.urls у нас будут содержаться все маршруты приложения women
создаем файл women.urls
импортируем модули from django.urls import path и from .views import *
прописываем список маршрутов
urlpatterns = [
    path('', index), # тоже самое что и http://127.0.0.1:8000/women/
    path('cats/', categories), # оже самое что и http://127.0.0.1:8000/women/cats/
    # префикс women/ мы прописали в файле coolsite/urls.py -->  path('women/', include('women.urls'))
]

и так как в path('women/', include('women.urls')) указали префикс 'women/' то его нужно
будет указывать везде и в браузерной строке
